feat: Add NPX and Python support for browser-native development

Implemented comprehensive development tools to support Spirit (Claude Code) workflows:

## New Features

### NPX - Execute npm Packages
- Dynamic ESM package loading from esm.sh CDN
- Two modes: library loading and inline execution (-e flag)
- Browser-native execution without server
- Tested with: nanoid, date-fns, lodash-es, preact, ms, chalk

### Python 3.11 - Full Python Support
- Pyodide WASM integration for browser-native Python
- Execute inline code (-c flag), files, and modules (-m flag)
- Full Python 3.11 standard library included
- Lazy loading (only loads on first use)

### PIP - Python Package Manager
- Install pure Python packages via micropip
- Pre-compiled WASM packages (numpy, pandas, matplotlib, scikit-learn)
- List installed packages
- Seamless Pyodide integration

## Implementation Details

### Modified Files
- src/devtools.js: Added npx, python, python3, pip commands (~320 lines)
- src/commands.js: Updated help text to include new tools

### Usage Examples

NPX:
```bash
npx nanoid                          # Load package, show exports
npx -e "const {nanoid} = await import('https://esm.sh/nanoid'); return nanoid()"
```

Python:
```bash
python --version                    # Python 3.11.3 (Pyodide)
python -c "print([x**2 for x in range(10)])"
python script.py
```

PIP:
```bash
pip install numpy
python -c "import numpy; print(numpy.__version__)"
```
